## Reporting vulnerabilities

**DO NOT open a public issue to report security vulnerabilities.**

### How to report

<!-- CUSTOMIZE -->
Send an email to: [security email]

Include:
1. Vulnerability description
2. Steps to reproduce
3. Potential impact
4. Affected version

### What to expect

- Acknowledgment within **48 hours**
- Impact assessment within **5 business days**
- Patch for critical vulnerabilities within **14 days**
- Researcher credit (if desired) after the fix

## Disclosure policy

We follow **responsible disclosure** — we do not disclose publicly until:
1. The vulnerability has been fixed
2. Users have had a reasonable amount of time to update

## Scope

### In scope
- API endpoints
- Authentication and authorization logic
- Input validation
- Sensitive data handling

### Out of scope
- Brute-force attacks without real impact
- Social engineering
- Physical attacks
- Vulnerabilities in third-party dependencies (report directly to the maintainer)
