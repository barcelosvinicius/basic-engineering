## Reportando vulnerabilidades

**NÃO abra uma issue pública para reportar vulnerabilidades de segurança.**

### Como reportar

<!-- CUSTOMIZAR -->
Enviar e-mail para: [email de segurança]

Incluir:
1. Descrição da vulnerabilidade
2. Passos para reproduzir
3. Impacto potencial
4. Versão afetada

### O que esperar

- Confirmação de recebimento em até **48 horas**
- Avaliação de impacto em até **5 dias úteis**
- Patch para vulnerabilidades críticas em até **14 dias**
- Crédito ao pesquisador (se desejado) após correção

## Política de disclosure

Seguimos **responsible disclosure** — não divulgamos publicamente até que:
1. A vulnerabilidade esteja corrigida
2. Os usuários tenham tido tempo razoável para atualizar

## Escopo

### Em escopo
- Endpoints da API
- Lógica de autenticação e autorização
- Validação de inputs
- Gestão de dados sensíveis

### Fora de escopo
- Ataques de força bruta sem impacto real
- Engenharia social
- Ataques físicos
- Vulnerabilidades em dependências de terceiros (reportar diretamente ao maintainer)
